\hmsubsection{User Requirements}{OAH}{UMA}

The user requirements were defined in collaboration with the client representatives
from USN and formalised in a requirements document agreed upon in March 2026. The
complete requirements matrix, including acceptance criteria, verification methods,
and priority classifications, is provided in Appendix~\ref{appendix:requirements}.

The requirements are structured into six categories: functional~(F), machine
learning~(ML), mechanical~(M), system~(S), non-functional~(NF), and test~(T). The
critical functional requirements cover ball detection ($\geq 95\,\%$ detection
rate), colour classification ($\geq 90\,\%$ accuracy), pick-up reliability
($\geq 90\,\%$ success rate), and correct container placement ($\geq 90\,\%$). The
system shall additionally complete at least ten consecutive sorting cycles without
manual restart, and handle fault conditions without producing uncontrolled movement.
Non-functional requirements specify operation across 300--800\,lux and a stop time
of $\leq 1$\,s upon a fault signal.
